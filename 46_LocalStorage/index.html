Local Storage

Local Storage is a web server object which are not sent to server with each request.

This data survives a full page refresh and even a full browser restart.

If you want to see local Storage in chrome then you go to web page inspect --> Application --> LocalStorage.


This are the methods provided by localStorage
  1. setItem(key,value) : localStorage.setItem("name","Adarsh")
    Store Key or Value pair.
  2. getItem(key) : localStorage.getItem("name")
    Get the value by key.
  3. removeItem(key) : localStorage.removeItem("name")
    remove the key with its value.
  4. clear() : localStorage.clear()
    Delete Everything.
  5. key(index) : localStorage.key(index)
    get key on a given position.
  6. length : localStorage.length()
    The number of stored items.
    7. we can get and set values like an object.

    localStorage.one = 1
    alert(localStorage.one)
    delete localStorage.one

    Important Note :

    1. Both key and values must be strings.
    2. We can use the two JSON methods to store objects in localStorage.
      • JSON.Stringify(object) --> converts objects to JSON Strings.
      • JSON.parse(string) --> converts strings to objects.(must be a valid JSON).

Session Storage

Session Storage acts in a similar way to localStorage. The Only difference here is that while data stored in local storage has an expiration date,
the data stored in Session storage will expire as soon as the page's session ends, that's it, When the page is closed.
Used less often than localStorage.Properties and methods are same as LocalStorage but:
  1. The sessionstorage exists only within the current browser tab Another tab with same page will have a different storage.
  2. The data survives page refresh, but not closing/opening the tab.

  1. setItem(key,value) : sessionStorage.setItem("name","Adarsh")
    Store Key or Value pair.
  2. getItem(key) : sessionStorage.getItem("name")
    Get the value by key.
  3. removeItem(key) : sessionStorage.removeItem("name")
    remove the key with its value.
  4. clear() : sessionStorage.clear()
    Delete Everything.
  5. key(index) : sessionStorage.key(index)
    get key on a given position.

Storage Event

When the data gets updated in localStorage or sessionstorage, Storage event triggers with these Properties:
  1. Key --> The key
  2. Old value --> Previous Value
  3. New value --> New Value
  4. Url --> Page URL
  5. Storage Area --> Local or sessionstorage
We can listen the onstorage event of window which is triggered when updates are made to the same storage from other documents. window.onstorage=(e)=>{ alert("changed") console.log(e) } Open Two pages of same web page.
